# 演示脚本

## 演示目标

用最短路径展示系统已经打通的核心闭环：

1. 语音输入
2. 语义识别
3. 日程创建、查询、修改、删除的确认流程
4. 提醒调度与 WebSocket 推送
5. TTS 语音播报

## 演示前准备

- 后端已启动
- 前端已启动
- PostgreSQL 和 Redis 已连接
- 浏览器允许麦克风权限
- 浏览器支持 Web Speech API 和 SpeechSynthesis 更好；不支持时也能展示手动输入和文字回复
- 建议使用固定测试用户 `user_id = u001`
- 建议保持同一个 `session_id` 完成一轮完整演示

## 演示顺序

### 场景 1：语音添加日程

操作：

1. 点击语音输入
2. 说出：

   ```text
   明天下午三点提醒我交项目文档
   ```

预期：

- 页面显示识别文本
- 后端返回创建结果
- 系统回复中出现自然语言确认
- 若开启 TTS，自动播报回复

### 场景 2：语音查询今天安排

操作：

1. 继续使用同一会话
2. 说出：

   ```text
   我今天有什么安排
   ```

预期：

- 页面展示当天安排列表或文字回复
- 日程按照时间排序播报
- 如果为空，显示“暂无安排”

### 场景 3：缺少时间时系统追问

操作：

1. 说出：

   ```text
   提醒我交项目文档
   ```

预期：

- 系统判断信息不完整
- 页面出现追问文案
- `conversation_state` 被保存
- 此时不执行真正的创建操作

### 场景 4：删除日程前确认

操作：

1. 说出：

   ```text
   删除明天的会议
   ```

预期：

- 如果匹配到唯一日程，页面进入删除确认状态
- 用户点击“确认”前不发生真实删除
- 若有多个候选，先让用户选择候选

### 场景 5：修改日程前确认

操作：

1. 说出：

   ```text
   把下周三的会改到周四下午两点
   ```

预期：

- 系统查找候选并生成修改草稿
- 页面展示修改确认文案
- 用户确认前不更新数据库

### 场景 6：到点提醒和语音播报

操作：

1. 创建一个 1 分钟后触发的提醒
2. 等待提醒到点

预期：

- WebSocket 收到 `reminder_triggered`
- 页面弹出提醒提示
- 对应事件高亮
- TTS 播报提醒内容
